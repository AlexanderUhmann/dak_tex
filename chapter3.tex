\chapter{Тестирование алгоритмов машинного и глубокого обучения на больших данных по кредитному скорингу}

В предыдущей главе было выяснено, что алгоритмы градиентного бустинга и нейронных сетей не имеют эффектов переобучения и недообучения. В случае градиентного бустинга используются гиперпараметры по умолчанию, результаты метрик которых не ниже метрик других алгоритмов. Это означает, что не требуется поиск гиперпараметров для данного алгоритма и можно остановиться на гиперпараметрах по умолчанию. Идентичная ситуация возникает у нейронной сети с отличием в том, что у нее метрики точности немного хуже, чем у алгоритма градиентного бустинга. В этой главе будут подобраны гиперпараметры для алгоритмов деревьев решений и случайного леса, поскольку они обладают эффектом переобучения. Основным инструментом для поиска оптимальных гиперпараметров используются инструменты $GridSearchCV$\footnote{\cite{GridSearchCV} (дата обращения: 25.11.2025). Загл. с экрана.} и $RandomizedSearchCV$\footnote{\cite{RandomizedSearchCV} (дата обращения: 25.11.2025). Загл. с экрана.}.

\section{Подбор гиперпараметров для алгоритмов машинного обучения при оценке возврата кредита}

Гиперпараметры -- это такие параметры модели, которые задаются до начала работы модели. Правильно подобранные параметры позволяют улучшить качество предсказаний моделей. Каждому алгоритму МО присуще собственные наборы гиперпараметров, которые будут рассматриваться ниже. Существуют два метода подбора гиперпараметров:

\begin{enumerate}[nosep]
	\item $GridSearchCV$ -- метод подбора оптимальных гиперпараметров для модели с помощью перебора всех возможных комбинаций из заданного набора, что является ресурсозатратным способом.
	\item $RandomizedSearchCV$ -- метод, позволяющий выбирать количество случайных комбинаций из заданного набора гиперпараметров. Является наиболее эффективным методом по времени при работе с большими данными.
\end{enumerate}

\subsection{Деревья решений}

Для алгоритма классификации решающих деревьев ($Decision\ Tree\ Classifier$) основными гиперпараметрами являются:

\begin{enumerate}[nosep]
	\item $criterion$ -- критерий качества разбиения, определяющая показатель, по которому алгоритм лучше классифицирует клиентов. \begin{enumerate}
	\item [a)] $gini$ -- индекс Джини\footnote{Джини К. (1884-1965 гг.) -- итальянский статистик, экономист, социолог и демограф.}, измеряющий неоднородность двух классов (0 и 1) внутри узла. Если индекс равен 0, значит в узле содержатся объекты одного класса. Чем индекс ближе к 0, тем лучше алгоритм определяет класс клиента. Индекс Джини в алгоритме $Decision\ Tree$ используется по умолчанию, поскольку быстрее вычисляется;
	\item [b)] $entropy$ -- энтропия Шеннона\footnote{Шеннон К.Э. (1916-2001 гг.) -- американский инженер, криптоаналитик и математик, заложил основы теории информации.}, измеряющий степень смешанности классов. Высокая энтропия свидетельствует о высокой неоднородности классов, чем ниже энтропия, тем узел более однородный. Алгоритм выбирает разбиения с наиболее низкой энтропией, т.е. содержатся большинство объектов одного класса;
	\item [ с)] $log\_loss$ -- логарифмическая функция потерь. В данном случае алгоритм не просто выбирает класс, а оценивает их вероятности. За каждый неверный прогноз вводит штраф.
	\end{enumerate}
%	\item $splitter$ -- параметр, задающий способ разбиения каждого узла дерева решений.
%	\begin{enumerate}[nosep]
%		\item [a)] $best$ -- алгоритм последовательно перебирает все возможные признаки и пороги, которые лучше всего разделяют клиентов по целевой переменной. При данном параметре требуется больше времени на обучение.
%		\item [b)] $random$ -- в каждом узле дерева рассматривается случайная часть разбиения, среди которых выбирается лучшее значение. За счет этого модель быстрее обучается, но качество разбиений может быть немного хуже, чем при $best$.
%	\end{enumerate}
	\item $max\_depth$ -- максимальная глубина дерева решений. Чем больше глубина, тем длиннее цепочка вопросов, чтобы выдать прогноз по клиенту. При небольших значениях алгоритм делает мало разбиений и недообучается, т.е. плохо объясняет данные и дает невысокую точность на обучающей и тестовой выборках. При высоких значениях алгоритм, наоборот, переобучается, запоминает все особенности обучающей выборки, но плохо классифицирует клиентов на тестовых данных.
	\item $min\_samples\_split$ -- параметр задает минимальное число наблюдений в узле, при котором возможно следующее разбиение на два дочерних. Если в узле наблюдений меньше заданного порога, то дальнейшее разбиение не выполняется, и узел становится конечным.
	\item $min\_samples\_leaf$ -- параметр с минимальным заданным количеством наблюдений после разбиения в каждом узле, что позволяет дереву не создавать слишком маленькие группы клиентов и служит инструментов против переобучения алгоритма.
	\item $max\_features$ -- параметр задает случайное количество признаков, среди которых алгоритм ищет оптимальное разделение.
	\begin{enumerate}[nosep]
		\item [a)] $sqrt$ -- в каждой вершине выбирается количество признаков, равное квадратному корню из общего числа признаков.
		\item [b)] $log2$ -- число признаков, равное двоичному логарифму от их общего количества.
	\end{enumerate}
	\item $random\_state = 42$ -- произвольное число\footnote{Почему именно «42» - широко известное и запоминающееся значение по умолчанию среди исследователей. Это ответ на главный вопрос жизни, Вселенной и всего сущего в романе Дугласа Адамса «Автостопом по Галактике».}, задаваемое программе, чтобы при каждом запуске программы конечный результат оставался неизменным. 
\end{enumerate}

\subsection{Нейронная сеть}

Для классификации клиентов используется однослойная нейронная сеть, состоящая из входного слоя размерности 19 признаков, одного скрытого слоя из 32 нейронов с функцией активации $ReLu$ и выходного слоя с функцией $sigmoid$, заложенного в основе формирования вероятности дефолта. Для нейронной сети основными параметрами обучения являются:
\begin{enumerate}[nosep]
	\item $epochs$ -- число эпох обучения нейронной сети. Параметр показывает, сколько раз модель обучается по всей обучающей выборке. Если число эпох слишком мало, то нейронная сеть будет недообучена и неспособна выявить закономерности в данных. В данном случае используется 20 эпох, поскольку является достаточным для обучения модели.
	\item $batch\_size$ -- размер пакета данных, на котором модель обучается за один шаг обучения. Например, $batch\_size$ = 128 означает, что на одном шаге модель обрабатывает 128 клиентов. 
	\item $Dense(32)$ -- скрытый слой нейронной сети, состоящий из 32 нейронов. Каждый нейрон формирует собственную комбинацию входных признаков, позволяющие модели выявлять скрытые зависимости между характеристиками клиента и вероятностью дефолта.
	\item $activation = ReLU$ -- функция активации скрытого слоя. Функция $ReLU$ преобразует отрицательные числа в ноль, а положительные значения остаются без изменения, что позволяет алгоритму выявлять наиболее значимые комбинации признаков и создавать более сложные зависимости между входными данными и целевым признаком.
	\item $Dense(1)$ -- выходной слой, состоящий из одного нейрона. В данной работе решается задача бинарной классификации, поэтому на выходе используется только 1 нейрон, который формирует конечную оценку клиента о принадлежности к классу 0 или 1.
	\item $activation = sigmoid$ -- функция активации выходного слоя. Сигмоидальная функция преобразует входные значения в диапазон (0; 1) и выводит вероятность дефолта.
	\item $optimizer = Adam$ -- оптимизатор, с помощью которого нейронная сеть обновляет свои веса в процессе обучения. После каждого шага обучения модель сравнивает свой прогноз с фактическим значением и корректирует веса связей между нейронами. 
	\item $leraning\_rate = 0.005$ -- шаг обучения модели. 
	\item $loss = mse$ -- функция потерь, которая показывает насколько прогнозы модели отличаются от фактических значений целевого признака. 
	\item $BinaryAccuracy$ -- метрика точности классификации. Показывает долю верно классифицированных наблюдений. Порог $threshold$ = 0.5 означает, что при вероятности выше 0.5 клиент относится к классу дефолтных, а ниже -- к классу недефолтных клиентов. 
\end{enumerate}

Нейронная сеть показывает высокое качество классификации на обеих выборках (см. Таблицу \ref{nn_final_res}). Метрики $precision$, $recall$, $f1-score$ показывают хорошее распрзнование недефолтных и дефолтных клиентов. Значения $accuracy$ = 0.98 и $ROC-AUC$ = 0.97 также свидетельствуют о хорошей разделяющей способности модели. Значения всех метрик практически совпадают, что говорит об отсутствии выраженного эффекта переобучения.

\section{Ограничения и возможности моделей}

На основе проделанной работы в рамках данного датасета можно выделить существенные ограничения и дальнейшие возможности работы моделей при прогнозировании кредитного риска заемщиков. Под ограничениями работы с моделями ИИ понимаются проблемы технического и нормативного характера. 

Во-первых, одной из ключевых проблем являются вычислительные ресурсы. Модели дерева решений, случайного леса, градиентного бустинга и нейронной сети требуют значительных вычислительных мощностей при обработке 3 млн данных клиентов банка, что увеличивает длительность обработки на одном алгоритме до нескольких дней. Подбор оптимальных гиперпараметров для улучшения прогностических способностей алгоритмов также может занимать время до недели. Если расширить масштаб анализа на всю банковскую систему Казахстана, где, по данным ПКБ, около 9 млн\footnote{\cite{24kz} (дата обращения: 6.03.2026). Загл. с экрана.} экономически активного населения имеют кредиты, то требования к вычислительным ресурсам возрастают многократно. Решением может быть либо улучшение IT-инфраструктуры, либо переход на облачные технологии.

Во-вторых, качество работы моделей зависит от экономической среды вместе с которой изменяется и потребительское поведение заемщиков. Поэтому модель, обученная на данных предыдущих периодов, может давать систематические ошибки в прогнозировании дефолта. Следовательно, необходимо регулярно обновлять данные кредитных историй и переобучать модели на обновленных данных.

В-третьих, для качественного прогноза модели помимо кредитных историй можно также анализировать альтернативные данные. Однако доступ к ним ограничен законом о банковской тайне (ст. 53 «О банках и банковской деятельности в Республике Казахстан») и законодательством о персональных данных, которые позволяют использование таких данныъ только с согласия самого клиента. Кроме того, у каждого банка накапливается лишь часть кредитной истории, поскольку кредиты одного заемщика могут быть взяты в разных банках. Поэтому невозможно объединить все кредитные истории со всех банков в единую систему для анализа, т.к. необходимо получить разрешение от всех клиентов. Таким образом, каждый банк вынужден разрабатывать собственные подходы к оценке кредитоспособности заемщиков на основе ограниченного массива доступных банку данных. Создание единой межбанковской системы на основе ИИ, которая могла бы анализировать полную долговую нагрузку клиента, ограничивается правовыми барьерами.

Под возможностями моделей ИИ понимается их предсказательная способность, характеризующаяся метриками точности. Следует отметить, что прогноз модели напрямую зависит от того, насколько корректно определено понятие аномального клиента. Например, в курсовой работе\footnote{см. Там же. С. 18.}, где исследовались данные кредитных историй по Республике Тайвань, метрики точности оказались ниже, чем в данной работе. По этой причине, следует внимательно относиться к предобработке данных, потому что качество их очистки во многом определяет конечный результат прогноза моделей ИИ.
